# VaultKeeper Design Context

## Users
A mix of legal professionals (prosecutors, defence lawyers, judges), field investigators, forensic analysts, and international court staff. They work in high-stakes environments handling criminal evidence — from war crimes to financial fraud. Sessions are long, tasks are detail-oriented, and errors have real consequences. Users range from tech-savvy investigators to less technical legal counsel. Multilingual (English, French).

## Brand Personality
**Authoritative. Precise. Trustworthy.**

VaultKeeper is the system of record for evidence that may be presented in court. It must feel like the interface itself would hold up under scrutiny — meticulous, reliable, and beyond question.

## Aesthetic Direction
**"Precision instrument with institutional weight"** — Think of the intersection between a Bloomberg terminal's density and a Swiss bank's restraint. The UI should feel like a high-end tool designed for professionals who care about accuracy, not aesthetics for aesthetics' sake. Every element should feel deliberately placed and purposeful.

- **Not playful** — no rounded bubbly cards, emoji, or casual copy
- **Not generic SaaS** — no dashboard templates with colored metric cards
- **Not dark-hacker** — no neon, no cyberpunk, no "cool" for cool's sake
- **Not government legacy** — modern, fast, and refined despite the serious domain

### References
- Bloomberg Terminal: information density, keyboard-driven, professional
- Linear: clean modern tool UI, fast, no clutter
- Stripe Dashboard: typography-led hierarchy, generous whitespace, precise alignment
- The Hague courtrooms: institutional seriousness, wood tones, gravitas

### Anti-references
- Generic admin dashboards (AdminLTE, template UIs)
- Playful SaaS (Notion's emoji-heavy style, Slack's casualness)
- Overly decorated enterprise software

## Theme
Both light and dark mode, respecting system preference. Light mode is the primary design surface — it should feel like working with legal documents. Dark mode for extended sessions.

## Design Principles

1. **Density without clutter** — Show information efficiently. Legal professionals need to see case details, evidence lists, and custody chains without excessive clicking. But never sacrifice readability for density.

2. **Trust through precision** — Pixel-perfect alignment, consistent spacing, and clear typography signal that the system itself is trustworthy. If the UI looks sloppy, users won't trust the custody chain.

3. **Hierarchy through typography, not decoration** — Use font weight, size, and spacing to create hierarchy. Minimize borders, shadows, and color-coding. Let the content speak.

4. **Progressive disclosure** — Start simple, reveal complexity through interaction. A case list is clean; drilling into a case reveals layers of detail. Don't overwhelm on first glance.

5. **Calm authority** — The interface should lower anxiety, not raise it. Neutral tones, deliberate animations, clear feedback. When handling sensitive evidence, the last thing users need is a flashy UI competing for attention.

## Color Direction
- Warm stone/slate neutrals (not cold grays)
- Deep navy or charcoal as the anchoring dark color
- A single accent color for interactive elements — subdued amber or teal (not blue — too generic)
- Status colors: muted earth tones (olive for active, amber for caution, brick for holds)
- Avoid: pure black, pure white, saturated primaries

## Typography Direction
- A refined serif or geometric sans for headings — something with character but not decorative
- A highly legible sans-serif for body/data — optimized for reading long text and scanning tables
- Monospace only for reference codes and hashes — never for UI chrome

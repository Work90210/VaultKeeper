Intake · SealingWhat ingest actually does

The door is not a web upload. It's a notarised border crossing.

Files cross the boundary through a resumable chunked transfer. Each 4 MB chunk is hashed in the browser, reconciled server-side, and countersigned by at least two of your witness nodes before the file manifest is committed. If reconciliation fails, the file is never written to disk in a usable form — we don't do partials.

01

Client-side hash

SHA-256 is computed in the investigator's browser before a single byte leaves their machine. The server compares against the hash it independently derives from the reassembled stream.

02

RFC 3161 timestamp

The combined manifest is sent to a configurable TSA pool (default: three geographically-diverse authorities). The returned token is stored alongside the hash.

03

Witness countersignature

Quorum of witness nodes — institutional peers you've federated with — each sign the manifest. The seal is valid only once ≥ 2 of ≥ 3 have signed.
